# 客服工作台项目分析文档

## 项目背景
基于阿里云客服工作台，我们需要构建一个智能客服系统，通过FAQ管理和相似度匹配来替代人工客服，实现用户提问的自动回答。

## 后端开发角色职责

### 数据库设计
**需要设计数据库**，主要包含以下表结构：

1. **类目管理表**
   - 一级类目表：存储主要业务分类
   - 二级类目表：存储具体的功能模块分类
   - 类目关系表：维护父子类目关联关系

2. **FAQ管理表**
   - FAQ主表：存储标准问题和答案
   - 相似问法表：存储同一问题的不同表达方式

3. **用户交互记录表**
   - 记录用户提问历史
   - 存储匹配结果和用户反馈

### API接口开发
- 类目增删改查接口
- FAQ录入和管理接口
- 相似度匹配查询接口
- 用户提问处理接口

## 算法开发角色职责

### 模型选择
**需要使用BERT模型**，具体选择：
- **bert-base-chinese**：适合中文文本处理
- 原因：相比传统机器学习方法，BERT能更好地理解语义，处理同义词和不同表达方式

### BERT使用方案
1. **文本编码**：将用户提问和FAQ问题转换为向量表示
2. **相似度计算**：通过余弦相似度计算向量间距离
3. **匹配排序**：找出最相似的FAQ条目返回给用户

### 大模型使用考虑
是否使用大模型取决于以下因素：
不需要大模型的情况：
- 问题库规模小
- 问题类型固定，变化不大
- 对响应速度要求高
- 成本预算有限

需要考虑大模型的情况：
- 问题复杂，需要理解上下文
- 需要生成新的回答而非匹配历史回答
- 有充足的预算

本项目暂时不需要使用大模型，因为问题库规模小，问题类型相对固定，变化不大。

## 技术架构
- 前端：用户界面展示
- 后端：Spring Boot + MySQL
- 算法层：Python + Transformers + BERT

后端负责数据存储和业务逻辑，算法负责智能匹配，共同构建完整的智能客服系统，也有可能后端与算法工程师是同一个人。
